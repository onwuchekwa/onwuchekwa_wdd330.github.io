Lesson Note

Programming languages, JavaScript inclusive, do not execute comments. The purpose of comments are to help the programmer remember the funtion or purpose for the lines of code; it also makes the program easy to read and understand. Data types, string, numbers, symbols, booleans, numbers, null, undefined, and object, are the foundation upon which JavaScript is built. The first six are primitive data types and the last is non-primitive.

The ES6 version of JavaScript recommends that const and let keywords should be used to declare variables. Variables that cannot be changed should be declared as const, whereas those that can change should be declared as let.

A few new ordered lists were introduced in ES6 to compliment Arrays. They include Sets and Maps. Arrays can be single or multidimensional. The key difference between an array, a set and a map is that set contains unique values, an array contains duplicate values, and a map uses the key-pair technique.

The flow of statements is controlled using the if and else statement. An if statement can be nested. However, switch statement is recommended in place of long if/else statements.

In addition to reducing the number of lines a programmer will write, a function makes it easier to reuse code, stop and start the execution of a program, provide an input value for other functions. Every function returns a value whether explicitly stated or not. When variables are placed in the parenthesis of a function during its declaration, those variables are referred to as parameters, but when they are provinded during a function call, they are known as arguments.
