Lesson Note

When data are entered incorrectly, it could cause our web applications not to work as expected and compromise the security of users' data and those of the application. Therefore, web designers needs to validate users' input to make sure it conforms to the type and format of data required.

There are two ways to validate data on a web application: client-side validation and server-side validation.Validation done in the browser is called client-side validation, while validation done on the server is called server-side validation. Built-in form validation and JavaScript are the two types of client-side validation.

Question

  • Why should people both the client-side and server-side validations?
  • What is the difference between built-in form validation and the use of JavaScript to validate forms?
